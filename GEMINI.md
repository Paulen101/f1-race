# Gemini Project Rules

## Context
- **Primary Language:** Python (for AI/Data) / JavaScript (for Web)
- **Frameworks:** YOLOv8/v11, FastAPI, React
- **Project Goal:** [Insert brief goal, e.g., Waste detection system or F1 Analytics]

## Coding Standards
- Use type hints for all Python functions.
- Prefer functional components and Hooks for React.
- Follow PEP 8 style guidelines for Python.
- Always include Docstrings for public-facing methods.

## Behavior
- When fixing bugs, explain the root cause before providing the code.
- If a library is outdated, suggest the modern equivalent.
- For data analysis tasks, prioritize efficiency (Vectorization over Loops).

## Git & Workflow
- Branch naming: `feature/brief-description` or `fix/issue-id`.
- Write concise, imperative commit messages (e.g., "Add person-counting logic").
